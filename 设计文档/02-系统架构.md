## 架构分层

### 前端分层

| 层 | 职责 | 关键文件 |
|---|------|---------|
| 视图层 (View) | UI 渲染、用户交互 | `App.tsx`, `pages/`, `components/` |
| 逻辑层 (Logic) | 回测状态管理、数据加载 | `hooks/useBacktest.ts` |
| 引擎层 (Engine) | 回测计算、策略执行 | `engine/strategies.ts`, `engine/portfolio.ts` |
| 数据层 (Data) | CSV 解析、数据缓存、基金列表 | `utils/csv.ts`, `data/etf-list.ts` |
| 类型层 (Types) | TypeScript 类型定义 | `types/index.ts` |

### 后端分层

| 层 | 职责 | 关键函数 |
|---|------|---------|
| 路由层 | HTTP 请求处理、参数校验 | `search_funds`, `fetch_fund`, `chat` |
| 业务层 | 基金列表缓存、数据拉取、AI 调用 | `get_fund_list`, `fetch_fund` |
| 数据源层 | 第三方数据接口调用 | `ak.fund_open_fund_info_em` |

## 关键设计决策

### 1. 回测引擎放在前端
回测计算在浏览器端执行（`engine/strategies.ts`），而非服务端。原因：
- 单基金回测计算量小（最多几千条数据），浏览器完全可胜任
- 无需等待网络请求，交互即时响应
- 无服务端计算资源压力

### 2. 双写 CSV 目录
净值 CSV 同时保存到 `src/data/prices/` 和 `public/data/prices/`：
- `public/` 目录在 Vite 构建时原样复制到 dist，确保生产环境可加载
- `src/data/prices/` 供开发时直接引用

### 3. 前端缓存策略
- 基金列表：服务端内存缓存 1 小时
- 净值数据 CSV：通过 `localStorage` 记录拉取时间，30 分钟内不重复拉取
- 组合回测结果：模块级变量缓存，切换 Tab 不丢失

### 4. API Key 安全
DeepSeek API Key 存储在前端 `localStorage`，不经过后端持久化。AI 对话请求由 FastAPI 代理转发。

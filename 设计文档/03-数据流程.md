# 数据流程

## 数据获取流程

```
用户操作                          系统行为
────────                          ────────
选择基金代码
    │
    ├── 前端检查 localStorage 最近拉取时间
    │       │
    │       ├── 30分钟内已拉取 → 直接使用本地 CSV
    │       │
    │       └── 需要刷新 → POST /api/fetch/:code
    │                           │
    │                    Python FastAPI
    │                           │
    │                    ak.fund_open_fund_info_em()
    │                    (东方财富天天基金网)
    │                           │
    │                    解析净值数据 + 格式转换
    │                           │
    │                    写入 CSV 文件 (双目录)
    │                           │
    │                    返回 {code, records, firstDate, lastDate}
    │                           │
    ├── fetch /data/prices/:code.csv
    │       │
    │       └── parseNavCSV() 解析为 NavRecord[]
    │
    └── 传入回测引擎 runBacktest()
```

## CSV 数据格式

### 原始数据源 (AkShare 返回)

| 字段 | 说明 |
|------|------|
| 净值日期 | 交易日日期 |
| 单位净值 | 基金单位净值 |
| 累计净值 | 基金累计净值 |
| 日增长率 | 当日涨跌幅 (%) |

### 输出 CSV 格式 (类 K 线)

```csv
date,open,close,high,low,volume,amount
2024-01-02,,1.2345,,,,
2024-01-03,,1.2401,,,,
```

- `date`: 日期 (YYYY-MM-DD)
- `close`: 单位净值（唯一有效数据字段）
- `open`, `high`, `low`, `volume`, `amount`: 保留为空，预留扩展

### 前端解析 (parseNavCSV)

```typescript
// 读取 close 列作为净值，过滤无效行，按日期排序
NavRecord[] = [{ date: string, nav: number }]
```

## 组合聚合的前向填充算法

每只基金的回测结果按各自定投日产生数据点，日期不完全相同。需要将不同基金的数据对齐到统一的日期序列上：

```
时间线:   D1    D2    D3    D4    D5
基金A:    ●           ●     ●
基金B:          ●           ●     ●

聚合后:
D1: A=●(有值),  B=◌(前向填充 0)
D2: A=●(沿用),  B=●(有值)
D3: A=●(有值),  B=●(沿用)
D4: A=●(有值),  B=●(有值)
D5: A=●(沿用),  B=●(有值)
```

使用 cursor 模式：遍历排序后的所有日期，若某基金在当前日期无数据，沿用上一次已知值。

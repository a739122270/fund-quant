# 回测引擎设计

## 概述

回测引擎是系统的核心计算模块，位于 `src/engine/` 目录下，包含两个文件：

- `strategies.ts` — 单基金回测核心算法
- `portfolio.ts` — 组合回测聚合逻辑

## 单基金回测 (strategies.ts)

### 入口函数

```typescript
function runBacktest(navData: NavRecord[], config: BacktestConfig): BacktestResult
```

### 核心状态变量

| 变量 | 含义 | 变化方式 |
|------|------|---------|
| `ts` (totalShares) | 当前持仓份额 | 买入 +, 卖出 - |
| `ti` (totalInvested) | 累计投入金额 | 每次买入累加 |
| `cb` (cashBack) | 卖出回笼现金 | 卖出时累加收益 |
| `cycB` (cycleBase) | 周期基准成本 | 卖出时按比例扣减 |
| `peakR` | 周期收益率峰值 | 用于移动止盈判断 |
| `lastM` | 上次定投的年月标记 | 防止同月重复定投 |

### 逐日模拟循环

每天依次执行以下逻辑：

```
for each 交易日:
  1. 计算周期收益率 (cycR)
     └── cycR = (持仓市值 - 周期成本) / 周期成本 × 100%

  2. 补仓判断 (Dip Buy)
     └── 若 cycR 低于阈值，按比例追加买入

  3. 定投判断 (DCA)
     ├── 每月/每周/每两周 在指定日期执行
     ├── 若开启智能定投，根据估值分位调整金额
     └── 一次性投资：仅首日执行

  4. 更新峰值收益率 (peakR)
     └── 用于止盈回撤判断

  5. 止盈判断 (Take Profit)
     └── 从峰值回撤达到阈值时，按比例卖出

  6. 止损判断 (Stop Loss)
     └── 亏损达到阈值时，按比例卖出

  7. 记录当日资产快照 (DailyValue)
     └── { date, invested, value, nav, cash }
```

### 交易记录生成

每笔买卖生成 TradeRecord：

```typescript
interface TradeRecord {
  date: string          // 交易日期
  nav: number           // 当日净值
  action: 'buy' | 'sell'
  investAmount: number  // 交易金额
  fee: number           // 手续费
  shares: number        // 交易份额
  totalShares: number   // 交易后持仓份额
  totalInvested: number // 交易后累计投入
  marketValue: number   // 交易后持仓市值
  return: number        // 交易后累计收益率
}
```

### 综合指标计算

回测结束后计算：

| 指标 | 计算方式 |
|------|---------|
| 累计收益率 | (最终资产 - 累计投入) / 累计投入 × 100% |
| 年化收益率 | (最终资产 / 累计投入)^(1/年数) - 1 |
| 最大回撤 | 从历史峰值到后续谷底的最大跌幅 |
| 年化波动率 | 日收益率标准差 × √252 |
| 夏普比率 | (年化收益 - 无风险利率) / 年化波动率（无风险利率取 2.5%） |

### 边界条件处理

- 回测区间至少需要 2 个交易日数据
- 定投模式下至少生成 1 笔交易记录
- 最大回撤计算时，总资产 ≤ 0 的点跳过
- 日收益率计算过滤掉起始资产为 0 的点

## 组合回测 (portfolio.ts)

### 入口函数

```typescript
async function runPortfolioBacktest(config: PortfolioConfig): Promise<PortfolioResult>
```

### 处理流程

1. **数据刷新**: 批量检查所有基金数据是否需要更新，并行拉取
2. **独立回测**: 每只基金独立执行 `runBacktest()`
3. **日期聚合**: 将所有基金的每日资产按日期对齐
4. **汇总指标**: 基于聚合后的组合资产序列计算综合指标

### 日期聚合算法

```
输入: fundResults[].result.dailyValues
输出: 统一的 dailyValues[] (组合层面)

1. 收集每只基金的 dailyValues 到 Map<date, value>
2. 合并所有日期 → 排序
3. 前向填充 (forward fill): 每个日期每只基金取最近已知值
4. 汇总: invested = Σ, value = Σ, cash = Σ
```

### 指标说明

组合层面的指标计算方式与单基金一致，区别在于 dailyValues 中的 value/cash 是多只基金的汇总值。

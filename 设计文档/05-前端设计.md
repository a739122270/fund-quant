## 状态管理

### 单基金回测 (useBacktest Hook)

```typescript
interface BacktestState {
  loading: boolean       // 是否正在计算
  refreshing: boolean    // 是否正在拉取最新数据
  error: string | null   // 错误信息
  result: BacktestResult | null  // 回测结果
}
```

状态通过 `useState` 在 App 组件管理，`useBacktest` hook 封装了 `run` 和 `clear` 方法。

### 组合回测 (PortfolioBacktest 组件内部状态)

| 状态 | 存储方式 |
|------|---------|
| initialCapital / monthlyAvailable | useState + localStorage |
| funds[] (基金列表) | useState + localStorage |
| selectedIdx (当前选中基金) | useState + localStorage |
| result (回测结果) | useState + 模块级变量缓存 |

关键设计：组合回测结果缓存到模块变量 `_cachedResult`，单基金回测和组合回测通过 Tab 切换时不丢失各自的回测结果。

## 策略配置面板 (ConfigPanel)

### 支持的策略类型

**基础策略：**
- **定期定额 (DCA)**: 按周/双周/月在指定日期定投固定金额
- **一次性投资 (Lump Sum)**: 区间首日全额买入

**增强策略（仅定投模式可用）：**

| 策略 | 触发条件 | 操作 |
|------|---------|------|
| 智能定投 | 净值处于历史低分位 | 买入更多 (150%) |
| | 净值处于历史高分位 | 买入更少 (50%) |
| 补仓策略 | 持仓亏损达阈值 | 追加买入 (按市值比例) |
| 止盈策略 | 从峰值回撤达阈值 | 部分卖出 |
| 止损策略 | 持仓亏损达阈值 | 部分卖出 |

每种增强策略支持 2 级阶梯触发。

### 配置持久化

每次参数变更实时保存到 localStorage，页面刷新后恢复。组合回测中每只基金独立保存各自的 `BacktestConfig`。

## 图表组件

### NavChart (净值走势图)
- X 轴：日期
- Y 轴左侧：累计投入 (柱状图)
- Y 轴右侧：持仓市值 (折线图)
- 展示投入与市值的对比关系

### ReturnChart (收益分布图)
- 展示每笔交易买入/卖出的收益情况

### FundContribution (组合贡献图)
- 展示组合中各基金的资产占比和贡献度

## AI 投资顾问 (AIChat)

### 功能
- 基于用户当前持仓和策略，提供分析建议
- 支持自由对话提问

### 交互流程
1. 用户输入问题
2. 前端将历史消息 + 持仓上下文 JSON 发送给 `/api/chat`
3. 后端构建系统提示词 + 转发到 DeepSeek API
4. 返回结果解析为 JSON（评分/优势/风险/建议/基金分析）
5. 前端以结构化卡片渲染

### 输出结构 (AnalysisData)

```typescript
interface AnalysisData {
  summary: string           // 一句话总评
  score: number             // 0-100 综合评分
  strengths: string[]       // 优势点 (最多3条)
  risks: RiskItem[]         // 风险点 (含等级)
  suggestions: string[]     // 改进建议 (最多3条)
  asset_allocation: string  // 资产配置评估
  fund_analysis: FundAnalysisItem[]  // 逐只基金分析
}
```

# 策略模块设计

## 策略体系

系统支持 1 种基础策略 × 4 种增强策略的自由组合。

```
基础策略
├── 定期定额 (DCA)
│   ├── 频率: 每周 / 每两周 / 每月
│   ├── 金额: 基础每期金额
│   └── 增强策略 (可选叠加):
│       ├── 智能定投 (Smart DCA)
│       ├── 补仓策略 (Dip Buy)
│       ├── 止盈策略 (Take Profit)
│       └── 止损策略 (Stop Loss)
│
└── 一次性投资 (Lump Sum)
    └── 仅首日买入，无增强策略
```

## 定期定额 (DCA)

### 逻辑

```
每月/每周/每两周 指定日期执行一次买入

买入:
  买入金额 = 基础每期金额
  手续费   = 买入金额 × 申购费率
  实际投入 = 买入金额 - 手续费
  买入份额 = 实际投入 / 当日净值
```

### 频率判断

| 频率 | 判断逻辑 |
|------|---------|
| 每月 | `d.getDate() >= dayOfMonth` 且月份不重复 |
| 每两周 | `d.getDay() === dayOfWeek` 且 `(getDate()-1)/7 取整为偶数` |
| 每周 | `d.getDay() === dayOfWeek` |

**按月定投的特殊处理**: 使用 `year*12 + month` 标记上次定投月份，判断当月是否已执行。用 `>=` 而非 `===` 避免非交易日（如周末）导致漏投。

## 智能定投 (Smart DCA)

### 原理
基于净值在历史区间内的分位数动态调整每期投入金额。

### 计算
```
回顾窗口 = 最近 756 个交易日 (约 3 年)
净值分位 = (当前净值 - 窗口最低净值) / (窗口最高净值 - 窗口最低净值) × 100

if 分位 < 低分位阈值 (默认 30%):
  投入 = 基础金额 × 低分位倍数 (默认 150%)
elif 分位 > 高分位阈值 (默认 70%):
  投入 = 基础金额 × 高分位倍数 (默认 50%)
else:
  投入 = 基础金额 × 中间倍数 (默认 100%)
```

### 设计意图
- 净值处于历史低位时加码买入 ("低估多投")
- 净值处于历史高位时减少买入 ("高估少投")
- 适合长期定投用户平滑成本

## 补仓策略 (Dip Buy)

### 原理
持仓出现大幅亏损时，追加买入以摊薄成本。

### 触发条件
```
周期收益率 cycR = (持仓市值 - 周期成本) / 周期成本 × 100%
if cycR ≤ -阈值%:
  追加金额 = 当前持仓市值 × 追加比例%
  执行买入
```

### 阶梯设计
支持 2 级阶梯触发，每级阈值只触发一次：

| 阶梯 | 默认阈值 | 默认追加比例 | 含义 |
|------|---------|-------------|------|
| 1 | 亏损 10% | 20% 市值 | 小幅下跌时小补 |
| 2 | 亏损 20% | 40% 市值 | 大幅下跌时大补 |

**防重复机制**: 使用 `Set<thresholdPercent>` 记录已触发的阈值，同一阈值只触发一次。

## 止盈策略 (Take Profit)

### 原理
从周期收益率峰值回撤一定幅度时，部分止盈卖出锁定利润。

### 触发条件
```
peakR = 历史周期收益率峰值
回撤幅度 = peakR - 当前 cycR
if 回撤幅度 ≥ 阈值%:
  卖出份额 = 当前持仓 × 卖出比例%
  卖出金额 = 卖出份额 × 净值 - 手续费
  现金回笼 = 卖出金额 (累加到 cashBack)
  扣减周期成本 = 按卖出比例扣减
```

### 阶梯设计
| 阶梯 | 默认阈值 | 默认卖出比例 | 含义 |
|------|---------|-------------|------|
| 1 | 回落 20% | 卖出 50% | 小幅回撤减半仓 |
| 2 | 回落 50% | 卖出 80% | 大幅回撤清大部分 |

### 周期成本机制
卖出时按比例扣减 `cycleBase`，确保下次周期收益率的计算基础正确。止盈和止损共用 `cycleBase`。

## 止损策略 (Stop Loss)

### 原理
亏损超过阈值时，部分止损卖出以控制风险。

### 触发条件
```
if cycR ≤ -阈值%:
  卖出份额 = 当前持仓 × 卖出比例%
  (其余与止盈相同)
```

### 阶梯设计
| 阶梯 | 默认阈值 | 默认卖出比例 | 含义 |
|------|---------|-------------|------|
| 1 | 亏损 10% | 卖出 50% | 小幅亏损止损一半 |
| 2 | 亏损 20% | 卖出 80% | 大幅亏损清大部分 |

## 策略执行顺序

每日执行顺序至关重要，影响周期收益率计算：

```
1. 计算周期收益率 (基于前一日持仓)
2. 补仓判断 (基于当前亏损)
3. 定投判断 (独立于收益)
4. 更新峰值收益率
5. 止盈判断 (基于从峰值的回撤)
6. 止损判断 (基于当前亏损)
7. 记录当日快照
```

### 顺序设计理由
- 补仓优先于定投：先利用亏损机会摊薄成本，再执行常规定投
- 止盈在止损之前：优先锁定利润而非割肉
- 止盈/止损在同一天：两者共用 cycleBase 扣减，同一天不会同时触发（第一个触发后 cycleBase 已变化）
